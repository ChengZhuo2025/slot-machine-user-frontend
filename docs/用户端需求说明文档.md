# 爱上杜美人 - 用户端 H5 应用需求说明文档

## 一、项目概述

### 1.1 产品定位

爱上杜美人是一个创新型主题酒店服务平台，通过智能硬件技术与传统酒店业相结合，为用户提供个性化的主题房间体验服务。平台集成了酒店预订、智能设备控制、电商购物、分销推广等多种功能，打造全方位的生活服务生态。

### 1.2 商业模式

- **B2B2C 模式**：与酒店合作，在酒店房间内部署智能设备，为终端用户提供增值服务
- **智能硬件集成**：通过智能柜体设备，实现扫码开柜、远程控制等功能
- **多元化收入**：房间预订费、设备使用费、商品销售、分销佣金等
- **会员体系**：通过会员制度提升用户粘性和消费频次

### 1.3 目标用户群体

- **主要用户**：18-45 岁成年用户，追求新奇体验和隐私保护
- **消费特征**：具有一定消费能力，重视服务质量和用户体验
- **使用场景**：情侣约会、商务休闲、朋友聚会等私密性场景
- **地域分布**：一二线城市为主，逐步向三四线城市扩展

### 1.4 产品价值

- **用户价值**：便捷的预订体验、私密的使用环境、丰富的商品选择
- **商户价值**：增加房间收入、提升服务差异化、扩大客户群体
- **平台价值**：构建完整的商业生态、获得多元化收入来源

## 二、功能架构设计

### 2.1 总体架构

```
用户端H5应用
├── 首页模块（酒店推荐、房型展示、会员服务）
├── 酒店服务模块（酒店列表、详情、预订）
├── 设备控制模块（扫码开柜、远程开锁）
├── 商城模块（商品浏览、购买、支付）
├── 分销模块（推广赚钱、佣金管理）
├── 用户中心模块（个人信息、订单、会员）
└── 基础服务模块（登录注册、支付、客服）
```

### 2.2 核心业务流程

1. **用户注册/登录** → **浏览酒店** → **选择房型** → **预订支付** → **到店使用** → **设备体验** → **商品购买** → **分享推广**

2. **扫码开柜流程**：扫描二维码 → 验证权限 → 发送开锁指令 → 设备响应 → 使用体验

3. **分销推广流程**：申请分销 → 获取推广链接 → 分享获客 → 获得佣金 → 提现收益

## 三、详细功能需求

### 3.1 首页模块

#### 3.1.1 页面头部

**功能描述**：展示应用品牌形象和核心入口

**具体要求**：

- **标题栏**

  - 品牌名称：爱上杜美人（支持自定义 Logo）
  - 右侧消息图标：显示未读消息数量，点击进入消息列表
  - 支持消息推送红点提醒功能

- **Banner 轮播图**
  - 支持 3-5 张图片轮播，自动播放间隔 3-5 秒
  - 支持手动滑动切换，显示页码指示器
  - 每张 Banner 支持跳转链接配置
  - 图片尺寸要求：750x300px，支持多分辨率适配

#### 3.1.2 酒店推荐区

**功能描述**：基于用户位置推荐最近的合作酒店

**具体要求**：

- **位置服务**

  - 自动获取用户地理位置（需用户授权）
  - 支持手动切换城市功能
  - 计算并显示酒店距离（精确到 0.1km）

- **酒店信息展示**
  - 酒店封面图（尺寸：200x150px）
  - 酒店名称（最多显示 20 个字符，超出显示省略号）
  - 详细地址（包含区域和街道信息）
  - 距离显示（如：12.7km）
  - 导航按钮（调用地图应用导航）
  - "更多"按钮（跳转至酒店列表页）

#### 3.1.3 热门房型推荐

**功能描述**：展示平台热门房型，支持快速预订

**具体要求**：

- **展示规则**

  - 显示 3 个热门房型，横向滑动查看
  - 基于预订量、评分等数据智能推荐
  - 优先显示用户附近的酒店房型

- **房型信息**
  - 房型封面图（尺寸：280x200px）
  - 房型名称（如：豪华大床房、情侣主题房）
  - 剩余房间数量（实时更新）
  - 房间介绍（如：90 分钟大床房+无限次互动体验）
  - 价格信息：门市价、会员价（会员价高亮显示）
  - 可预订时间（如：11:30 可约，支持时段显示）
  - "立即预订"按钮（醒目设计，引导用户行动）

#### 3.1.4 会员服务区

**功能描述**：展示会员权益和开通状态

**具体要求**：

- **未开通状态**

  - 显示会员特权介绍（专享价格、优先预订等）
  - 开通优惠信息
  - "立即开通"按钮

- **已开通状态**
  - 会员等级标识
  - 会员有效期显示
  - 专享特权说明
  - 续费提醒（到期前 7 天）

#### 3.1.5 附近酒店区域

**功能描述**：分类展示附近合作酒店

**具体要求**：

- **分类展示**

  - Tab 标签分类：全部、电竞酒店、主题酒店、私享酒店、尊住酒店
  - 支持左右滑动切换标签
  - 每个分类下展示对应类型酒店

- **酒店卡片信息**
  - 酒店封面图（尺寸：150x120px）
  - 酒店名称
  - 价格区间（如：¥199-399）
  - 剩余房间数量
  - 点击卡片跳转酒店详情页

#### 3.1.6 优惠信息区

**功能描述**：展示用户可获取的优惠信息

**具体要求**：

- **布局设计**：一行三列等宽布局
- **优惠券展示**

  - 50 元新人体验券（新用户专享）
  - 100 元满减券（消费门槛设置）
  - 显示券面额、使用条件、有效期

- **包月卡展示**
  - 包月卡权益介绍
  - 价格和优惠幅度
  - 立即购买按钮

#### 3.1.7 精选优品区

**功能描述**：推荐平台精选商品

**具体要求**：

- **展示布局**：2 列布局，共展示 6 个商品
- **商品信息**
  - 商品图片（尺寸：150x150px）
  - 商品标题（最多 2 行显示）
  - 价格信息：普通价格、会员价格
  - 加入购物车按钮
  - 立即购买按钮

#### 3.1.8 申请入口区

**功能描述**：为潜在合作伙伴提供申请入口

**具体要求**：

- **申请运营按钮**

  - 备注文字：酒店方申请入驻运营
  - 跳转至运营申请表单页面

- **申请代理按钮**
  - 备注文字：我想代理这个品牌
  - 跳转至代理申请表单页面

#### 3.1.9 客服支持

**功能描述**：提供便捷的客服联系方式

**具体要求**：

- 右下角固定位置浮动按钮
- 客服图标+文字设计
- 点击可选择：在线客服、电话客服、帮助中心

#### 3.1.10 底部导航

**功能描述**：应用主要功能入口导航

**具体要求**：

- **导航菜单**：首页、商城、扫一扫、赚钱、我的
- **实现方式**：不使用 UniApp 原生 tabBar，采用自定义底部导航组件
- **设计要求**
  - 中间"扫一扫"图标放大突出显示
  - 未选中状态为灰色，选中状态为主题色（紫色）
  - 使用 MynaUI 图标集中的相应图标
  - 支持图标+文字组合显示
  - 固定在页面底部，不随滚动移动
  - 支持页面切换动画效果

### 3.3 设备控制模块

#### 3.3.1 扫码开柜功能

**功能描述**：通过扫描二维码实现设备开锁

**实现流程**：

1. **扫码识别**

   - 调用相机权限扫描二维码
   - 解析二维码获取设备信息
   - 验证二维码有效性和权限

2. **权限验证**

   - 验证用户登录状态
   - 检查用户使用权限
   - 确认设备使用资格

3. **开锁控制**
   - 发送开锁指令到设备
   - 实时显示开锁状态
   - 处理开锁成功/失败情况

**页面设计**：

- 扫码界面带取景框和引导文字
- 支持手电筒开关功能
- 支持从相册选择二维码图片
- 扫码历史记录查询

#### 3.3.2 编码开锁功能

**功能描述**：支持手动输入设备编码开锁

**实现方式**：

- 编码输入界面（支持数字键盘）
- 编码格式验证
- 与扫码开锁相同的权限验证流程
- 支持常用编码收藏功能

#### 3.3.3 远程开锁功能

**功能描述**：通过网络远程控制设备开锁

**应用场景**：

- 线下核销码使用
- 客服远程协助开锁
- 特殊情况应急开锁

**安全机制**：

- 多重身份验证
- 操作日志记录
- 异常行为监控

## 四、技术实现要求

### 4.1 技术架构

- **前端技术**：Vue.js 3.x + UniApp 3.0
- **UI 组件库**：UniApp 内置组件 + 自定义组件
- **状态管理**：Pinia 状态管理
- **路由管理**：UniApp 页面路由
- **HTTP 库**：UniApp request API
- **图标库**：Iconify + MynaUI 图标集
- **动画库**：Animate.css
- **移动端适配**：UniApp 响应式单位，多端适配

### 4.2 兼容性要求

- **多端支持**：H5、微信小程序、App（iOS/Android）
- **浏览器支持**：iOS Safari 12+、Android Chrome 70+、微信浏览器
- **屏幕适配**：使用 UniApp 的响应式单位，支持多种屏幕尺寸
- **系统版本**：iOS 12+、Android 8.0+、微信版本 7.0+
- **网络环境**：2G/3G/4G/5G/WiFi 网络优化

### 4.3 性能要求

- **页面加载**：首屏加载时间<3 秒（3G 网络环境）
- **操作响应**：页面切换响应时间<500ms
- **图片优化**：支持 WebP 格式，懒加载机制
- **缓存策略**：合理利用浏览器缓存和本地存储

### 4.4 安全要求

- **数据传输**：HTTPS 加密传输
- **用户认证**：JWT Token 认证机制
- **敏感信息**：支付相关信息加密处理
- **接口安全**：防重放攻击、接口签名验证

## 五、用户体验设计

### 5.1 界面设计原则

- **简洁性**：界面简洁清晰，信息层次分明
- **一致性**：设计风格统一，交互方式一致
- **易用性**：操作简单直观，学习成本低
- **美观性**：色彩搭配协调，视觉效果良好

### 5.2 交互设计要求

- **导航设计**：清晰的导航结构，用户能快速找到目标功能
- **反馈机制**：操作后及时给出反馈，包括成功、失败、加载状态
- **容错设计**：对用户的错误操作给出友好提示和修正建议
- **无障碍设计**：支持语音朗读，字体大小调节

### 5.3 视觉设计规范

- **色彩系统**：

  - 主色调：#8B5CF6（紫色）
  - 辅助色：#EC4899（粉红色）、#F472B6（浅粉红）
  - 中性色：#333333（深灰）、#666666（中灰）、#999999（浅灰）

- **字体规范**：

  - 主要字体：苹方/思源黑体
  - 标题字号：18px-24px
  - 正文字号：14px-16px
  - 说明字号：12px

- **图标规范**：

  - 图标库：MynaUI 图标集（通过 Iconify 集成）
  - 导航图标：home、shopping-cart、scan、wallet、user 等
  - 功能图标：search、filter、location、phone、message 等
  - 图标尺寸：16px-32px（根据使用场景调整）

- **间距规范**：
  - 页面边距：16px
  - 模块间距：24px
  - 元素间距：8px-16px

### 5.4 动效设计

- **动画库**：使用 Animate.css 提供的动画效果
- **页面切换**：使用 slideInRight、slideOutLeft 等页面过渡动画
- **加载状态**：使用 pulse、spin 等加载动画
- **按钮反馈**：使用 bounceIn、pulse 等交互反馈动画
- **列表操作**：使用 fadeInUp、slideInLeft 等列表操作动效
- **弹窗动画**：使用 zoomIn、fadeIn 等弹窗显示动画

## 十、风险评估与应对

### 10.1 技术风险

- **设备兼容性风险**：不同型号智能设备的接口差异

  - 应对：制定统一的设备接入标准，预留扩展接口

- **第三方服务依赖**：支付、地图等第三方服务的稳定性

  - 应对：选择多个服务提供商，建立备用方案

- **性能瓶颈风险**：高并发情况下的系统稳定性
  - 应对：充分的压力测试，弹性扩容机制

### 10.2 业务风险

- **用户接受度**：新业务模式的市场接受程度

  - 应对：小范围试运营，根据反馈调整产品策略

- **法律合规性**：相关法律法规的变化

  - 应对：持续关注政策变化，确保业务合规

- **竞争压力**：同类产品的竞争
  - 应对：持续产品创新，提升用户体验

### 10.3 项目风险

- **需求变更**：开发过程中的需求调整

  - 应对：建立需求变更控制流程，评估影响

- **进度延期**：开发进度滞后风险

  - 应对：合理的时间缓冲，并行开发策略

- **人员流失**：关键开发人员离职
  - 应对：知识文档化，团队备份机制
